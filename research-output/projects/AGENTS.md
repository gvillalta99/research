# AGENTS.md

## Escopo
Armazena outputs por projeto, espelhando os submódulos de `../../projects/`.

## Estrutura esperada
Ao iniciar output para um projeto, criar `research-output/projects/<project-slug>/` com subpastas conforme necessário:
- `literature-review/`
- `analysis/`
- `drafts/`
- `reviews/`
- `final/`
- `assets/`

## Regras
- O nome de `<project-slug>` deve coincidir com o nome do submódulo em `../../projects/`.
- Outputs intermediários e finais ficam aqui, não dentro do submódulo, salvo necessidade explícita do projeto.
- Cada pasta de projeto deve ter um `INDEX.md` simples quando começar a acumular material.
